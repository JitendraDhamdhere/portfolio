Industrial sectors and building enterprises operating in Canada are subject to extremely strict safety and environmental hazard protocols. Verifying, logging, and tracking contractor compliance documents is crucial for minimizing institutional liabilities and passing municipal and federal audits.

I designed and developed the core backend workflow for a robust **Contractor Management & Hazard Documentation System** for a Canadian enterprise. The Engineering Challenge

Managing industrial contractor compliance is historically a paperwork bottleneck. Contractors must submit multi-role insurance, safety declarations, and hazard reports. Validating these forms through multi-step checklists manually is highly error-prone, and failing to block contractors with expired credentials from entering active construction zones introduces heavy regulatory fines.

Furthermore, when a contractor completes document audits successfully, the system must issue secure, tamper-proof compliance certificates. Generating these certificates as high-quality PDF files dynamically on the server and storing them securely without running out of disk space or causing performance drops during bulk hiring seasons poses a major scalability challenge.

My Implementation Role

I engineered the multi-step verification workflows, mapped database compliance schemas, configured Spring Security user-level filters, and built the server-side PDF generator.

  • Multi-Step Compliance Wizard

    I designed a transactional document verification checklist engine in Java. When a contractor uploads their insurance and safety profiles, my backend validates file types, tracks approval states (Submitted, In-Review, Action-Required, Validated), and blocks access key allocations until all checklist stages resolve to 'Validated'.

  • iText5 PDF Generation Pipeline

    I built a server-side document compiler using the iText5 library. When a compliance check completes, the system pulls student/contractor profile parameters, binds them to a custom-designed vector template, compiles a compact, secure PDF certificate, and uploads the document as an encrypted byte stream.

  • Multi-Role Security Gateways

    I configured user authorization boundaries utilizing Spring Security. By matching roles (Contractor, Auditor, Global Administrator) to specific class-level endpoints, I ensured that contractors can only write to their own profiles, while auditors have broad cross-auditing access.

  • Optimized Database Registry

    I mapped the relational schemas for safety log registries and document checklists in MySQL. By applying indexing parameters across foreign keys (such as contractor ID and submission date), I optimized document tracking and verification queues.
