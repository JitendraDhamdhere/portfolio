The Engineering Challenge

In diesel logistics, live inventory tracking is highly volatile. When a customer places a delivery request, the system must verify available stock in nearby diesel tankers (active drivers) dynamically. Standard transactional systems cause lock delays or duplicate assignments if parallel dispatch requests access the exact same driver ledger simultaneously.

Furthermore, dispatch algorithms must prioritize driver routes in real-time. If a driver is currently completing a delivery, the system must smoothly queue secondary assignments without dropping active connection sockets or causing transaction isolation failures in the operational ledger.

My Implementation Role

I engineered the secure logistics APIs, designed the normalized database tables, set up the JWT authorization filters, and built the live inventory tracking logic.

  • JWT Secure API Gateway

    I designed the security boundary using Spring Security and JSON Web Tokens. Mobile client apps and administrative portals obtain short-lived encrypted JWT credentials, validated at the filter-chain level, protecting operational logs and active logistics routes against unauthorized requests.

  • Automated Dispatch Matcher

    I developed a location-conscious driver dispatch allocation service in Java. By querying spatial MySQL coordinate tables, the engine matches pending orders with nearby active driver tanks, initiating dispatch workflows asynchronously without blocking user request threads.

  • Live Inventory Sync Manager

    I designed a transactional controller with optimistic locking (`@Version` annotations in Hibernate). This safeguards the system against parallel assignment overlaps: if two admins attempt to dispatch to the same driver simultaneously, one transaction commits successfully while the other resolves gracefully without corrupting stock totals.

  • State Machine Order Tracker

    I engineered a strict logistics state machine (Ordered → Assigned → In-Transit → Dispensing → Fulfilled). Each transition updates active ledgers, validates security tokens, and synchronizes real-time status details directly to customer and admin dashboards.
