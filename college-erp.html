The Engineering Challenge

An educational ERP system is highly complex because of the diverse user access roles (Admins, HR Directors, Librarians, Faculty, Guard, and Students) that interact with the exact same database. If access parameters are not locked down systematically, sensitive financial profiles, student contact registries, or grading indices could leak or be edited unauthorized.

Furthermore, campus administrators require fast, on-demand reporting capabilities. Converting large data sets containing hundreds of employee attendance logs or library book inventories into formatted Microsoft Excel reports programmatically without blocking main application execution threads poses a major scalability challenge.

My Implementation Role

I engineered the security routing architecture, mapped the core database model structures, and built the dynamic reporting and automated badge generation engines.

  • Custom Role-Based Access Control (RBAC)

    I designed a robust, annotation-driven RBAC layer utilizing Spring Security and JWT. I mapped granular authorization gates to specific endpoint classes, guaranteeing that only authenticated security guards can log gate entries, and only authorized HR directors can access salary variables.

  • Asynchronous Excel Reporting Engine

    I built a high-performance export module using Apache POI. To prevent main server thread blocks during large data runs, I implemented asynchronous execution parameters. The backend compiles the spreadsheet data in separate threads, returning a cached downloadable URL link to the Angular frontend on complete.

  • Student ID Badge Generator

    I programmed an automated ID card rendering loop. When a new student profile is verified by the registrar, my backend captures the photo byte array, parses the student details, and generates a formatted, print-ready PDF card directly using the iText5 rendering engine.

  • Modular Library & Security Database Mapping

    I designed the logical database structure for our security gate logs and inventory indexes. By indexing key query columns in MySQL (such as student badge numbers and book ISBNs), I optimized lookup search durations under high read volumes.
